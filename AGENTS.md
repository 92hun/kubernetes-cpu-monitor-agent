# Agent Routing

Read `ANALYSIS_AND_PLAN.md` first. It routes to five bounded plan files under `plans/`.

Rules:

1. Read and execute only the current numbered plan.
2. Do not open the next plan until the current plan has fresh PASS evidence.
3. Record status and evidence in the current plan before moving on.
4. Keep all project files inside this directory.
5. Never claim completion without fresh test, build, manifest, rollout, and log evidence.

Hard constraints are defined in `ANALYSIS_AND_PLAN.md`.
